### 1. 事务操作
```
事务流程：
    开启事务
    [异常 回滚事物]
    提交事务

# 查看事务
select @@autocommit;

# 开启事务
法一：设置事务提交方式
set @@autocommit=0;		0表示手动提交，1表示自动提交

法二：开启事务
start transaction/begin;

# 提交事务
commit;

# 回滚事务
rollback;

# 事务的四大特性
原子性：不可分割，要么成功，要么失败
一致性：事务完成，则数据状态一致
隔离性：隔离机制，保证事务操作独立
持久性：提交回滚，对数据永久改变

# 并发事务问题
脏读：一个事务读到另一个事务还没提交数据
不可重复读：一个事务限号读取同一条记录，但两次读取的数据不同
幻读：一个事务按照条件查询数据时，没有对应的数据行，但在插入数据时，又发现数据行存在

# 隔离级别              脏读   不可重复读   幻读
read uncommitted        1       1       1   读未提交
read committed          0       1       1   读已提交
repeatable read         0       0       1   可重复读(默认)
serializable            0       0       0   串行化

# 查看隔离级别
select @@transacition_isolation;

# 设置事务隔离级别
set [session/global] transaction isolation level 隔离级别;
```
