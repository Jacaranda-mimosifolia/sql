### 1. Mysql体系结构
```
连接层：连接上层的客户端，连接处理、授权认证、服务安全

服务层：sql接口、分析、优化、查询、函数

引擎层：负责数据的存储和提取，服务器可以通过API和存储引擎通信

存储层：将数据存储到文件系统，并与存储引擎交互
```

### 2. 存储引擎
```
存储引擎是基于表的数据存储、建立索引、更新、查询，是一种表类型

# 查看Mysql中所有的存储引擎
show engines;

# 创表指定存储引擎
create table t1(
    id varchar(10),
    name varchar(10)
)engine = myisam;


# InnoDB:可靠性、高性能，作用于行级锁、外键、事务，新版默认存储引擎
# 文件后缀:.idb
# InnoDB逻辑存储结构:
TableSpace:  表空间
Segment：    段
Extent：     区    1M
Page：       页      16K
Row：        行

# MYisam:不支持事务、不支持外键、支持表锁、访问速度快，早期默认存储引擎
# 文件后缀：
.sdi：存储表结构  json
.myd：存储数据   二进制
.myi：存储索引   二进制

# Memory：数据存储在内存中，断电丢失，表结构存储在外存，快
# 文件后缀：
.sdi
```
